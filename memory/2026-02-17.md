### Decision: FULL MIGRATION TO OPENCLAW (No n8n)

**Final Architecture:**
```
Master Agent (Sam): Routes calendar/contact requests
├─ calendar-agent: Calls Google Calendar API directly
├─ contact-agent: Calls Airtable API directly
└─ Plugin: Handles button clicks (SQLite state)
```

**Why full migration:**
- Cleaner than hybrid (no webhook bridge)
- More flexible (natural language routing)
- Better state management (SQLite)
- OpenClaw agents can call APIs directly with credentials

### Button Handling Solution

**Hybrid approach (VERIFIED DOABLE):**
```
Simple buttons (instant):
├─ cs| (calendar toggle) → Plugin updates SQLite + sends buttons (50ms)
├─ cc| (contact select) → Plugin updates SQLite + sends buttons (50ms)
└─ cc_nav| (pagination) → Plugin updates SQLite + sends buttons (50ms)

Complex buttons (LLM):
├─ cs|proceed → Agent decides → Creates calendar event
├─ cc|submit → Agent decides → Confirms contact
└─ cm|merge → Agent decides → Merges contacts
```

**Why it works:**
- Plugin runs BEFORE LLM (can intercept callbacks)
- Returns {handled: true} to skip LLM for simple ones
- Agent only gets involved for complex decisions
- No duplication, no conflicts
